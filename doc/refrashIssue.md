# 사용자 데이터 저장 문제

### 옵션

| 저장 방식          | 특징           | 비고                 |
| -------------- | ------------ | ------------------ |
| LocalStorage   | 새로고침 유지      | PC방 환경에서는 흔적 남아 위험 |
| SessionStorage | 새 탭/창 닫으면 삭제 | 새로고침은 유지됨          |
| 쿠키             | 새로고침 유지      | API 키 노출 X         |
| DB             | 가장 안전        | 토이 프로젝트면 과함        |

### 결론

로그인 없고 토이 프로젝트 기준:
→ **SessionStorage** 또는 **URL 파라미터/해시 기반 임시 저장**이 가장 적합
(PC방에서 개인정보 남을 가능성 최소화)
